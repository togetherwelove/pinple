# 명단 경로 전환 및 로그인 프리뷰 V1

## 기준 문서

- `../인원 리스트 자동 그룹화 서비스 시스템 프롬프트 및 PRD V2.pdf`
- 2026-07-12 사용자 라우팅 및 로그인 화면 결정

## 라우팅

- 공식 작업 공간 경로는 `/rosters`다.
- `/rosters`는 최근 명단을 기본 선택하고, `/rosters?view=create`는 기존 명단 선택을 건너뛰어 새 명단 작성 화면을 표시한다.
- 데스크톱 및 모바일의 `새로운 명단` 링크는 `/rosters?view=create`를 사용한다.
- 루트, OAuth 콜백, 명단 생성 후 이동, 명단 선택 및 삭제 후 이동, 보호 경로 검증은 모두 `/rosters`를 사용한다.
- 기존 `/dashboard`는 외부에 남은 이전 링크를 위해 `/rosters`로 리다이렉트만 수행한다.

## 인증

- `/rosters`는 인증되지 않은 요청을 `/login`으로 리다이렉트한다.
- 로그인 페이지는 이미 로그인된 사용자를 즉시 `/rosters`로 리다이렉트한다.

## 로그인 화면

- 스플릿 레이아웃을 사용한다.
- 데스크톱 왼쪽에는 정적 명단·조 편성 프리뷰를, 오른쪽에는 서비스 가치와 Google OAuth CTA를 둔다.
- 모바일에서는 로그인 영역을 먼저 표시하고 프리뷰는 그 아래에 배치한다.
- 프리뷰는 정적 UI이므로 실제 사용자 명단이나 조 편성 결과를 조회하거나 수정하지 않는다.

## 검증 기준

1. `/rosters`의 인증되지 않은 요청은 `/login`으로 이동한다.
2. 로그인과 OAuth 콜백 뒤에는 `/rosters`로 이동한다.
3. `/dashboard`는 `/rosters`로 리다이렉트한다.
4. 로그인 화면은 모바일과 데스크톱에서 CTA와 프리뷰가 겹치지 않는다.
5. 기존 명단이 있어도 `새로운 명단`을 누르면 새 명단 작성 화면으로 전환된다.
