# 명령 버튼 가시성 정책 V1

## 기준 문서

- 2026-07-12 사용자 질의응답
- `implementation-decisions-v2.md`

## 확정된 정책

| 명령 | 항상 표시 | 비활성 조건 |
| --- | --- | --- |
| 새 프로젝트 시작 | 예 | 프로젝트 이름이 비어 있음 |
| 명단 저장 | 예 | 입력 폼이 비어 있음 |
| 그룹별 정원 입력 | 예 | 저장된 명단이 없음 |
| 자동 그룹화 | 예 | 정원 합계가 전체 인원과 다름 |
| Excel 내보내기 | 예 | 그룹 결과가 없음 |

비활성화된 명령은 숨기지 않으며, 선행 작업을 설명하는 짧은 안내 문구를 함께 표시한다.

## 검증 기준

1. 빈 프로젝트에서도 모든 명령 버튼을 확인할 수 있다.
2. 선행 데이터가 생기면 해당 버튼이 다시 활성화된다.
3. 그룹 결과가 없을 때 내보내기는 실행되지 않는다.

## 검증 결과

- 명령 버튼과 그룹별 정원 입력은 조건부 렌더링 대신 `disabled` 상태로 처리한다.
- 비활성 원인을 각 작업 영역의 안내 문구로 표시한다.
- `tsc --noEmit` 통과
- `pnpm lint` 통과
- `pnpm build` 통과
