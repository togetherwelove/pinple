## 전략

| 식별자 | UI 이름 | 분배 규칙 |
| --- | --- | --- |
| `even` | 기본 골고루 섞기 | 성별 버킷을 나이 오름차순으로 처리한다. 정확히 같은 나이 안에서는 Fisher-Yates 셔플을 적용하고, 나이가 가까운 버킷부터 snake-direction 슬롯에 분배한다. 각 슬롯 행의 그룹 순서도 셔플해 동일 명단의 결과가 매번 달라지게 한다. 이 값은 두 체크박스가 모두 해제됐을 때 적용되며 UI에는 노출하지 않는다. |
| `age_similar` | 나이 비슷한 사람끼리 | 전체 명단을 나이 오름차순으로 정렬한다. 정확히 같은 나이 안에서만 셔플한 뒤, 지정된 각 그룹 정원만큼 순차 배정한다. |
| `gender_separated` | 같은 성별끼리 | 남성 우선과 여성 우선 배치를 각각 계산한다. 혼합 그룹 수가 적은 결과를 선택하고, 동률이면 혼합 그룹의 소수 성별 인원이 적은 결과를 선택한다. 따라서 성별은 가능한 한 같은 그룹으로 묶이고, 정원 제약상 필요한 경우에만 혼합된다. 나이에는 우선순위를 두지 않는다. |
| `gender_age_similar` | 같은 성별, 같은 나이끼리 | 성별 분리를 우선한 뒤, 각 성별 안에서 정확히 같은 나이를 우선으로 묶고 남은 인원은 가까운 나이순으로 배정한다. 성별 분리의 정원 제약으로 혼합 그룹이 필요할 때도 각 성별의 연령 순서를 유지한다. |

## UI 선택 규칙

- `나이 비슷한 사람끼리`와 `같은 성별끼리`만 체크박스로 노출한다.
- 둘 다 해제하면 `even`을 적용한다.
- 나이 체크박스만 선택하면 `age_similar`을 적용한다.
- 성별 체크박스만 선택하면 `gender_separated`을 적용한다.
- 둘 다 선택하면 `gender_age_similar`을 적용한다.

## 결과 저장

- 새 GroupResult JSON은 `groups`와 `strategy`를 함께 저장한다.
- 기존 GroupResult JSON에 전략 값이 없으면 `even`으로 해석한다.
- 드래그 앤 드롭으로 결과를 수정할 때도 생성 당시의 전략 값을 유지한다.

## 검증 기준

1. UI에는 두 체크박스만 보이고, 모두 해제된 기본값은 `even`이다.
2. 각 전략은 지정된 그룹 정원을 정확히 충족한다.
3. `even` 전략은 같은 명단으로 반복 실행 시 다른 그룹 조합을 만들 수 있다.
4. 결과 생성과 드래그 앤 드롭 저장 뒤에도 전략 값이 GroupResult JSON에 남는다.
5. 두 체크박스를 동시에 선택하면 `gender_age_similar` 결과가 저장된다.
