## 확정된 구현 정책

| 항목 | 결정 |
| --- | --- |
| 세션 종료 방식 | 클라이언트 컴포넌트에서 `@supabase/ssr` 브라우저 클라이언트의 `auth.signOut()`을 호출한다. |
| 성공 후 경로 | `router.replace("/login")`과 `router.refresh()`로 로그인 화면으로 이동한다. |
| UI 위치 | 데스크톱 사이드바 최하단의 계정 정보 영역과 모바일 프로젝트 탐색 영역에 로그아웃 버튼을 노출한다. |
| 계정 정보 원본 | 서버에서 확보한 Supabase `User`의 `user_metadata.full_name`, `user_metadata.name`, `email`을 사용한다. |
| 표시 우선순위 | 이름은 `full_name` -> `name` -> 이메일 순으로 표시하고, 이메일이 없으면 사용자 ID를 보조 정보로 표시한다. |
| 아바타 | 외부 이미지 설정에 의존하지 않도록 표시 이름의 첫 글자를 이니셜 아바타로 사용한다. |
| 사이드바 구조 | 데스크톱 사이드바는 세로 Flexbox로 구성한다. 프로젝트 탐색은 남는 높이를 사용하고 계정 영역은 `mt-auto`와 상단 테두리로 최하단에 고정한다. |
| 오류 처리 | 세션 종료 실패 시 버튼 인근에 오류 메시지를 표시하고 현재 화면을 유지한다. |
| 뒤로 가기 방어 | 기존 `src/proxy.ts`의 요청 단위 `auth.getUser()` 검증을 유지한다. 로그아웃 뒤 `/dashboard` 요청은 로그인 화면으로 리디렉션되어야 한다. |

## 검증 기준

1. 로그아웃 버튼은 대시보드에서만 표시된다.
2. 로그아웃 뒤 `/login`으로 이동한다.
3. 브라우저 뒤로 가기 또는 직접 `/dashboard` 요청은 유효한 세션이 없으면 `/login`으로 이동한다.
4. 데스크톱 사이드바 최하단에 현재 사용자의 이니셜, 이름, 이메일과 아이콘형 로그아웃 버튼이 보인다.

- `src/proxy.ts`는 `/dashboard`를 matcher에 포함하고, `auth.getUser()` 결과가 없으면 `/login`으로 리디렉션한다.
- `tsc --noEmit` 통과
