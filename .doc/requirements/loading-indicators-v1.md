# 비동기 작업 로딩 표시 V1

## 기준 문서

- `../인원 리스트 자동 그룹화 서비스 시스템 프롬프트 및 PRD V2.pdf`
- 2026-07-12 사용자 로딩 표시 요청

## 공통 구성

- `components/Spinner.tsx`는 Lucide `LoaderCircle`과 Tailwind `animate-spin`을 사용한다.
- 스피너는 장식용이며, 버튼의 텍스트가 작업 상태를 함께 전달한다.

## 작업별 상태

| 작업 | 상태 | 버튼/표시 | 중복 방지 |
| --- | --- | --- | --- |
| 새 명단 생성 | `isCreatingProject` | `명단 만드는 중...` | 생성 버튼 비활성화 |
| 텍스트 파싱 및 명단 저장 | `isSavingRoster` | `저장 중...` | 저장 버튼 비활성화 |
| CSV/Excel 읽기 | `isUploadingFile` | `불러오는 중...` | 파일 입력 비활성화 |
| 자동 조 편성 | `isGrouping` | `조 편성 중...` | 실행 버튼 비활성화 |
| 명단 삭제 | `isDeleting` | `삭제 중...` | 모달의 취소·삭제 버튼 비활성화 |

## 페이지 로딩

- `/rosters/loading.tsx`는 서버 데이터 조회 중 명단 입력과 조 결과 영역의 스켈레톤을 표시한다.

## 검증 기준

1. 각 작업 시작 시 해당 컨트롤에 스피너와 상태 문구가 나타난다.
2. 작업이 끝나거나 실패하면 해당 컨트롤이 다시 활성화된다.
3. 대시보드 서버 데이터 조회 중 `loading.tsx` 스켈레톤이 표시된다.
