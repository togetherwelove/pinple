## 삭제 규칙

- 사용자에게 보이는 용어는 `명단`이지만 내부 모델과 API 경로는 기존 `Project`, `/api/projects`를 유지한다.
- `DELETE /api/projects/[projectId]`는 현재 Supabase 사용자를 확인하고 `requireOwnedProject`로 소유권을 검증한다.
- Prisma `project.delete`를 사용한다. 현재 스키마의 `Person.project`와 `GroupResult.project` 외래 키는 모두 `onDelete: Cascade`이므로 인원과 조 편성 결과도 함께 영구 삭제된다.
- 삭제 전 다음 경고가 포함된 확인 모달을 표시한다: `정말 이 명단을 삭제하시겠습니까? 포함된 인원과 조 편성 결과가 모두 영구적으로 삭제됩니다.`
- 취소하면 API 요청을 하지 않는다.
- 현재 보고 있는 명단을 삭제하면 `/dashboard`로 이동한다. 다른 명단을 삭제하면 `router.refresh()`로 목록을 다시 불러온다.
- 삭제 실패는 버튼 인근 오류 메시지로 표시한다.

## 검증 기준

1. 소유 명단의 삭제 요청은 200 응답과 함께 명단, 인원, 조 편성 결과를 삭제한다.
2. 다른 사용자의 명단은 삭제할 수 없다.
3. 데스크톱과 모바일 명단 목록에서 삭제 모달을 열 수 있다.
4. 현재 명단 삭제 뒤 대시보드의 남은 명단 또는 빈 상태로 이동한다.
