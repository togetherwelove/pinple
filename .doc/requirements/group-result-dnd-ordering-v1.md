## 동작 규칙

- 같은 그룹의 카드 위에 드롭하면 해당 카드 위치로 멤버 순서를 변경한다.
- 다른 그룹의 카드 위에 드롭하면 해당 카드 위치에 멤버를 삽입한다.
- 그룹의 빈 영역에 드롭하면 해당 그룹의 마지막 위치에 멤버를 추가한다.
- 드래그 중인 카드는 DragOverlay로 표시하고, 나머지 카드는 sortable transform으로 즉시 이동을 보여 준다.
- DnD 컨텍스트 ID는 고정해 서버와 클라이언트의 접근성 속성이 달라지는 hydration 경고를 방지한다.
- 변경된 그룹 배열은 먼저 로컬 상태에 반영하고 Group Result PATCH API로 저장한다. 저장 실패 시 이전 배열로 되돌리고 오류를 표시한다.

## 검증 기준

1. 같은 그룹에서 카드 순서를 변경하면 화면과 저장 결과의 순서가 모두 바뀐다.
2. 다른 그룹의 카드 위 또는 빈 영역으로 이동하면 대상 위치에 정확히 삽입된다.
3. API 저장 실패 시 이전 순서와 그룹 배치로 복구된다.

## 검증 기록

- 2026-07-12: 로컬 `부산캠프` 30명 테스트 데이터에서 같은 그룹 내 카드 재정렬 후 새로고침으로 저장 상태를 확인했다.
- 2026-07-12: 다른 그룹의 카드 위 삽입을 확인하고, 테스트 종료 후 원래 15명/15명 구성과 순서로 복구했다.
- 2026-07-12: 고정 DnD 컨텍스트 ID 적용 뒤 새로고침에서 새 hydration 오류가 발생하지 않는 것을 확인했다.
- 2026-07-12: 고정 ID 적용 전 `tsc`, ESLint, Next.js 프로덕션 빌드는 통과했다. 적용 후 정적 검사는 Codex 실행 한도로 재실행하지 못했다.
