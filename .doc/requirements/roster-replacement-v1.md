# 명단 편집 및 보존 결정 기록 V1

## 기준 문서

- `implementation-decisions-v2.md`
- 2026-07-12 사용자 질의응답

## 확정된 정책

| 항목 | 결정 |
| --- | --- |
| 명단 입력 폼 복원 | 프로젝트 진입 시 저장된 Person 목록을 `이름, 성별, 나이` 형식으로 폼에 표시한다. 저장값 `M`, `F`는 각각 `남`, `여`로 표시한다. |
| 저장 방식 | 폼의 유효한 전체 명단을 해당 프로젝트 Person 목록으로 원자적으로 덮어쓴다. 폼에서 제거한 행은 Person 목록에서도 삭제한다. |
| 중복 행 | 데이터베이스 복합 고유 제약에 따라 동일한 `이름 + 성별 + 나이` 행은 한 건으로 저장한다. |
| 그룹 결과 보존 | 명단을 교체해도 기존 GroupResult JSON 스냅샷은 삭제하거나 수정하지 않는다. 기존 그룹 결과와 최신 명단은 동시에 보존된다. |
| 재그룹화 | 최신 Person 목록을 기준으로 새 GroupResult를 추가한다. |

## 검증 기준

1. 저장 후 새로고침하거나 프로젝트를 다시 선택해도 명단 입력 폼에 저장된 명단이 표시된다.
2. 행 삭제와 수정은 Person 목록에 반영된다.
3. 명단 교체 후에도 기존 그룹 결과는 그대로 표시된다.

## 검증 결과

- Person 삭제, 생성, Project 갱신은 하나의 Prisma 트랜잭션으로 처리한다.
- GroupResult에는 삭제 또는 갱신 쿼리를 실행하지 않는다.
- `tsc --noEmit` 통과
- `pnpm lint` 통과
- `pnpm build` 통과
