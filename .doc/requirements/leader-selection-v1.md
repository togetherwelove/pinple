## 데이터 규칙

- GroupResult `members.groups[].members[]`의 각 멤버는 `isLeader: boolean` 상태를 가진다.
- 새 조 편성 결과는 모든 멤버의 `isLeader` 값을 명시한다.
- 기존 결과에 `isLeader`가 없으면 `false`로 해석한다.
- 한 조에는 최대 한 명의 조장만 허용한다.

## 자동 선출 규칙

| 값 | UI 이름 | 규칙 |
| --- | --- | --- |
| `none` | 선출 안 함 | 모든 멤버의 `isLeader`를 `false`로 한다. 기본값이다. |
| `random` | 무작위 선출 | 각 조에서 한 명을 무작위로 지정한다. |
| `oldest` | 연장자 우선 선출 | 최고령 멤버를 지정한다. 최고령자가 여럿이면 그 안에서 무작위로 한 명을 지정한다. |

## 수동 변경과 이동 규칙

- 일반 멤버의 조장 지정은 같은 조의 기존 조장을 즉시 해제하고 선택한 멤버를 조장으로 지정한다.
- 현재 조장은 `조장 해제` 액션으로 조장 없는 조를 만들 수 있다.
- 조장을 다른 조로 이동했는데 대상 조에 조장이 이미 있으면, 저장 전에 선택 대화상자를 표시한다.
- 대화상자 A는 대상 조장을 유지하고 이동한 사람의 조장 권한을 해제한다.
- 대화상자 B는 이동한 사람을 대상 조장으로 지정하고 기존 대상 조장을 해제한다.
- 대화상자 선택 뒤에만 로컬 상태와 Group Result PATCH API를 갱신한다. 저장 실패 시 이전 상태로 되돌린다.

## 검증 기준

1. 자동 조 편성 시 선택한 방식에 따라 조장 상태가 저장된다.
2. 각 조에 조장이 두 명 이상 저장되지 않는다.
3. 카드에서 조장 지정과 해제가 즉시 반영되고 저장 실패 시 복구된다.
4. 조장 충돌 이동 시 A/B 선택 이후에만 결과가 변경된다.
