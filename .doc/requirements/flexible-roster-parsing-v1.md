- `../인원 리스트 자동 그룹화 서비스 시스템 프롬프트 및 PRD V2.pdf`
- 2026-07-12 사용자 결측치 처리 요청 및 질의응답

## 저장 및 마이그레이션 정책

- `Person.age`는 `Int?`로 변경하고, 나이가 없으면 `null`을 저장한다.
- `Person.gender`는 `UNKNOWN` 값을 저장할 수 있도록 `VARCHAR(7)`로 확장한다.
- 현재 데이터의 `age` 값은 모두 유효한 정수이므로 nullable 전환은 데이터 삭제 없이 적용한다.
- 기존 성별 `M`, `F`는 그대로 유지하고, 누락된 신규 입력에만 `UNKNOWN`을 저장한다.

## 휴리스틱 파싱 정책

- 줄바꿈으로 나눈 한 줄은 한 명의 데이터다. 빈 줄은 무시한다.
- 성별은 단독 토큰 `남`, `여`, `남자`, `여자`, `M`, `F`를 대소문자 구분 없이 인식한다. 없으면 `UNKNOWN`이다.
- `20대` 같은 연령대는 최솟값인 `20`으로 저장한다. 두 자리 숫자는 해당 나이로 저장한다. 둘 다 없으면 `null`이다.
- 성별·나이 토큰 및 구분 기호를 제거한 나머지를 이름으로 사용한다.
- 이름이 비어 있으면 원본 줄 번호를 포함한 `이름 미입력 {lineNumber}`을 자동 생성한다.

## 결측치 조 편성 정책

- `UNKNOWN` 성별은 독립 버킷으로 취급해 균등 분배한다.
- 나이 `null`은 각 성별 버킷의 나이 정렬 마지막에 배치한다.
- 성별 우선 전략에서도 `UNKNOWN` 버킷을 먼저 각 조에 균등 배정하고, 남성·여성은 각 조의 남은 정원에 배정한다.

## 검증 기준

1. 순서가 다른 성별·나이·이름 줄을 정상 저장한다.
2. 성별 또는 나이가 없더라도 해당 줄을 저장한다.
3. `UNKNOWN` 및 `age: null` 인원이 있는 조 편성이 정원 수를 유지한다.
4. 기존 `M`/`F` 및 나이 입력 결과는 계속 동작한다.
