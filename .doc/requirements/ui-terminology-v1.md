# UI 용어 변경 결정 기록 V1

> 2026-07-13 이후 최상위 작업 단위의 UI 표기는
> `project-multiple-group-results-v1.md`의 `프로젝트` 정책을 우선 적용한다.

## 기준 문서

- `../인원 리스트 자동 그룹화 서비스 시스템 프롬프트 및 PRD V2.pdf`
- 2026-07-12 사용자 용어 변경 요청 및 범위 정정

## 확정 범위

- 사용자에게 보이는 UI 텍스트, placeholder, 알림 메시지, 다운로드 기본 파일명에서 `프로젝트`를 `명단`으로 바꾼다.
- 사용자에게 보이는 UI 텍스트, 알림 메시지, 다운로드 시트명과 파일명에서 `그룹`을 `조`로 바꾼다.
- 새로 생성되는 조 이름은 `1조`, `2조` 형식으로 저장한다. 기존 결과의 `그룹 N` 이름은 화면과 내보내기에서 `N조`로 표시한다.
- Prisma 모델, 데이터베이스 테이블과 필드, API 경로, TypeScript 타입과 변수명은 변경하지 않는다.
- API 오류 메시지는 화면에 노출될 수 있으므로 UI 용어 정책에 따라 `명단`과 `조`로 바꾼다.

## 검증 기준

1. 데스크톱 및 모바일 탐색에서 `프로젝트` 대신 `명단`이 표시된다.
2. 새 항목 생성 화면과 오류 메시지에 `명단`이 표시된다.
3. API 경로와 DB 스키마에는 변경이 없다.
4. 화면과 내보내기에서 `그룹`이 아닌 `조`와 `N조` 표기가 사용된다.

## 검증 기록

- 2026-07-12: `app/src`의 TypeScript 및 TSX 사용자 노출 문자열을 검색해 `프로젝트`, `새 프로젝트`, `프로젝트 이름`, `조 편성`, `조장` 표기가 남아 있지 않음을 확인했다.
- 2026-07-12: Prisma 스키마, API 경로와 내부 식별자는 이 변경 범위에서 수정하지 않았다.
- 2026-07-12: 로컬 대시보드에서 데스크톱 사이드바의 `명단`, `새로운 명단` 표기와 기존 `그룹` 표기를 확인했다.
- 2026-07-12: 로컬 대시보드에서 `조 설정`, `자동 조 편성`, `조 편성 결과`, `1조` 형식의 기존 결과 표시를 확인했다.
