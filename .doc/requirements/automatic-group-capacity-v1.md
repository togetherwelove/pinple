# 자동 그룹 정원 계산 결정 기록 V1

> 2026-07-19 이후 이 문서의 수동 조 개수·정원 정책은 폐기하고
> `dynamic-group-structure-v1.md`의 현재 조 수 기반 전원 균등 편성을 적용한다.

> 2026-07-13 이후 프로젝트별 복수 조 편성 결과에서는
> `project-multiple-group-results-v1.md`의 동일 정원 및 나머지 대기 명단 정책을 우선 적용한다.

## 기준 문서

- `implementation-decisions-v2.md`
- 2026-07-12 사용자 질의응답

## 확정된 정책

| 항목 | 결정 |
| --- | --- |
| 계산 시점 | 저장된 명단 인원 수 또는 목표 그룹 수가 바뀔 때 자동 계산한다. |
| 배분 방식 | 전체 인원을 목표 그룹 수로 균등 분배하고 나머지는 앞 그룹부터 하나씩 더한다. 예: 5명, 3그룹은 `2, 2, 1`이다. |
| 수동 조정 | 자동 계산된 각 그룹 정원은 사용자가 직접 수정할 수 있다. |
| 버튼 | `기본 정원 만들기` 버튼은 제거한다. |
| 자동 그룹화 | 정원 합계가 전체 인원과 같을 때 바로 활성화한다. |

## 검증 기준

1. 명단 저장 뒤 기본 목표 그룹 수에 맞는 정원이 자동 표시된다.
2. 목표 그룹 수를 바꾸면 정원이 즉시 다시 계산된다.
3. 정원 입력을 수동으로 바꾸면 자동 그룹화는 합계가 맞을 때만 활성화된다.

## 검증 결과

- 목표 그룹 수 입력 이벤트에서 균등 정원을 계산한다.
- 명단 인원 수 변화 시 Workspace를 새로 만들어 기본 정원을 다시 계산한다.
- `기본 정원 만들기` 버튼을 제거했다.
- `tsc --noEmit` 통과
- `pnpm lint` 통과
- `pnpm build` 통과
